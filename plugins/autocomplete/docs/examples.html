REDCap Autocomplete Plugin Examples

Using the REDCap Autocomplete Plugin, you can narrow the item selection of a list. As you type, the list is filtered to display only those items that match. You can then select one of the items from the list and return that value to the form.


Instructions

Note: The following examples are intended to serve as a template so you don't have to start from scratch. They will probably not work "as-is". In order to get these examples to work, you must:

  1. Already have installed the REDCap Autocomplete Plugin code in the document root of your REDCap server.

  2. Be logged into a REDCap project or have an active session running before using the plugin.

  3. Already have created a stand-alone REDCap Project that will serve to store the autocompletion item matches. See the Example Data Dictionary and Data Export section below.

  4. Modify the examples below to match the field variable names and project ids of your installation as specified in REDCap Autocomplete Plugin documentation.

  5. Note: Your code must not contain duplicate <div> id's!


REDCap Autocompletion with One Variable

The Autocomplete Plugin allows the user to perform matching on list items of data that are coming from another REDCap project. The selected item is the name returned to the REDCap project form.

REDCap Autocompletion with Two Variables

The Autocomplete Plugin allows the user to perform matching on one database field (e.g. physician_name) and when the user selects a list item, the value that is returned to the REDCap project field is a different database field (e.g. study_id). When the user selects a name from the autocompletion list, the ID number associated with that name is returned to the REDCap project form.
